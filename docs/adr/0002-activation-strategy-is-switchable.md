# Activation strategy is switchable

Fontbrew initially activates installed packages by creating symlinks from a Fontbrew-owned activation directory into its managed store. The activation layer must remain strategy-based so Fontbrew can switch to copying font files if macOS or font tooling compatibility problems appear. In the current MVP, `symlink` is the only supported user-configurable strategy; `copy` is reserved in the model and rejected by config until copy activation is implemented. Fontbrew only manages activation artifacts inside its own activation directory and does not touch user-installed fonts outside that boundary.
